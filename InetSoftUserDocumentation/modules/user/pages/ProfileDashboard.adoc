How to enable Dashboard profiling in the Portal to measure execution time and identify performance bottlenecks. Profiling is enabled via the Account button > Profile option, which causes a 'Profiling...' link to appear below the Dashboard toolbar. The Profile dialog breaks down execution time and allows grouping by Cycle Name, Dashboard, Logical Model, Data Worksheet, or Component. A 'Show Detail Records' option provides a tabular view, and results can be exported.
endif::ai[]

You can profile the execution of your Dashboards to determine which aspects of execution consume the most time. To do this, follow the steps below:

. Press the {buttonAccount} at the top of the Portal and select {buttonProfile}.
+
image::user:Profiling1.png[alt="press Setting and select Open with Profiling"]
+
When this option is enabled, Dashboards will display a ‘Profiling...’ link below the toolbar.
+
image::Profiling2.png[alt="Profiling link appears below toolbar"]

. Press the ‘Profiling...’ link to open the ‘Profile’ dialog box. This displays a break-down of execution time.

. From the ‘`Group By`’ menu, choose  the way you want to break out the data: ‘`Cycle Name`’, ‘`Dashboard`’, ‘`Logical Model`’, ‘`Data Worksheet`’, or ‘`Component`’.
+
image::Profiling3.png[alt="choose Group By option"]
. Enable ‘Show Detail Records’ link to view a tabular display of the execution information.
+
image::Profiling4.png[alt="enable Show Detail Records"]

. *_Optional:_* Press the {buttonExport} to export the data.

. Press btn:[OK] to dismiss the dialog box.
