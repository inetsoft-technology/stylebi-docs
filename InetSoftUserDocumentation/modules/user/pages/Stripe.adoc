How to connect to Stripe as a data source in InetSoft. Steps: select the Stripe data source and press Create; enter a Name (label) for the connection; enter the API Key; optionally press Refresh Metadata if the data source structure has changed; optionally configure multi-tenant user settings; press OK. Verify the connection status icon — a check icon means success, an error icon means the connection failed (check settings or use EditConnection to correct them). After a successful connection, proceed to define a data set using QueryStripe.

. Select the ‘Stripe’ data source, and press btn:[Create]. This opens the data source definition screen.
+
image::dataSourceStripe1.png[alt="Stripe data source definition screen"]

. In the ‘Name’ field, enter a label for this data source connection.

. Enter the ‘API Key’.
+
. *_Optional:_* To refresh meta-data for the data source (which may be necessary if the data source's fields or structure have changed), press the btn:[Refresh Metadata] button.

. *_Optional:_* {multiTenantUser}

. Press btn:[OK].
+

. Verify that the data source you added is shown with a {buttonCheckIcon}, indicating that the connection was successful.
+
image::dataSourceStripe3.png[alt="connection successful icon"]
+
If the database is shown with an {buttonError}, this means that the connection was not successful.
+
image::dataSourceStripe4.png[alt="connection error icon"]
+
In this case, check the settings to make sure they are correct for your database. See xref:EditConnection.adoc[] for information on how to change connection settings.

After you have created a connection to the database, you can proceed to define a data set. See xref:dataworksheet:QueryStripe.adoc[].
