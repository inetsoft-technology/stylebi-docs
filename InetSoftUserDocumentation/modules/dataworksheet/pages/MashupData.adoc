*Data Worksheet* is StyleBI`'s tool for mashing up data. "`Mashing up data`" means  acquiring data from one or more data  sources, and then combining, transforming, and processing those data sets in a variety of ways to produce a desired result.  The final result is generally used to feed data to a Dashboard.

== When to Use
This approach is especially useful in the following cases:

[square]
* You are creating a data set from a *data model*. (In this case, you do not need to define any joins because all  joins are already defined in the data model.)
* You need to combine data from different data sources.

To create a data set in the User Portal by mashing up data, follow the steps below:



. Log into the User Portal.
+
include::ROOT:partial$createVisualizationStep2Scratch.adoc[]

. Press the {buttonCreate} at the top  of the User Portal, and select the {buttonWorksheet} option.
+
image::newWorksheet2.png[alt="press Create button at top of User Portal and select Worksheet"]
+
This opens Visual Composer in a separate browser tab and displays the ‘New Data Worksheet’ dialog box.
+
image::newWorksheet3.png[alt="New Data Worksheet dialog box showing Mashup Data and other options"]

. Choose the {buttonMashupData} option. This creates a blank Data Worksheet named ‘Untitled-X’, where X is an auto-incremented integer.
+
image::visualComposer.png[alt="blank Data Worksheet canvas opened in Visual Composer after choosing Mashup Data"]

. Proceed to add  data blocks to the Data Worksheet. Drag the desired data (attributes, fields, or tables) from the left panel.
+
image::transform1.png[alt="drag data attributes or tables from left panel into the worksheet canvas"]
+
You can drag fields from any entity of the *same data model* into a single data block without creating explicit joins.  (The joins are already defined in the data model.)

. *_Optional_*: You can also drag a data block (or individual columns of a data block) from another *Data Worksheet*. In this case, the inserted data block remains linked to the original Data Worksheet in which it is defined, and updates automatically in response to changes in the source Data Worksheet. To disable this auto-update behavior, press the {buttonAutoUpdate} in the toolbar of the bottom panel.
+
image::auto-update1.png[alt="press Auto Update button in toolbar to disable automatic update from source worksheet"]
+
You can then force an update *on-demand* by right-clicking the data block (or by pressing the {buttonActions}) and selecting the ‘Update Mirror’ option. To restore the original auto-update behavior, press the {buttonAutoUpdate2}.


. *_Optional_*: To modify the properties of a column in a data block, such as its name, visibility, position, sort order, see xref:dataworksheet:ModifyColumn.adoc[].
+
. *_Optional_*: To view the query represented by the data block, see xref:dataworksheet:PreviewQuery.adoc[].  To preview the data returned by the data block, see xref:dataworksheet:PreviewData.adoc[].

. Mark your final result as the *primary data block*. To do this, right-click the data block and select ‘Set as Primary’ from the context menu.  {rightClickOptions} This makes the data block visible to  other Data Worksheets, and places it first in the list of data blocks available to any Dashboard that uses this Data Worksheet as its data source.
