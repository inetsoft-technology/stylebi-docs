To create a new data set from an existing OData  connection, follow the steps below:

. If you have not yet configured the database connection, see xref:user:OData.adoc[]. Then proceed with the steps below.

+
include::ROOT:partial$createVisualizationStep2QueryCreate.adoc[]

. Open the Data Worksheet that will contain the data. (See xref:dataworksheet:CreateNewWorksheet.adoc[] or xref:dataworksheet:EditDataWorksheet.adoc[].)

. Press the {buttonObject}, and select {buttonODataQuery}. This opens the ‚ÄòOData  Query‚Äô dialog box.

+
image:dataSourceOData5.png[alt="OData Query dialog box for configuring the data query"]


. Select an entity in the ‚ÄòEntity‚Äô menu. (To access *unbound* functions, *do not* select an entity. See below.)

. *_Optional_*: Enter a '`Function`'  to query (e.g.,  `GetNearestAirport`).
+
NOTE: If you have selected an entity in the '`Entities`' menu, the '`Function`' menu lists the available *bound* functions for the selected entity. If no entity is selected, the menu lists the *unbound* functions.


. *_Optional:_*  Select ‚ÄòExpand Arrays‚Äô to expand a record that contains an array into multiple records.

. Enter a comma-separated list in the ‚ÄòSelect‚Äô field (e.g., ‚ÄúCompanyName, ContactName‚Äù) to restrict the columns in the result set to the specified fields.  See https://docs.oasis-open.org/odata/odata/v4.0/errata02/os/complete/part2-url-conventions/odata-v4.0-errata02-os-part2-url-conventions-complete.html#_Toc406398163[System Query Option $select] in the OData documentation for information about the `$select` operator.
+
image:dataSourceOData6.png[alt="Select field with comma-separated column list entered"]

. Enter a filtering condition (e.g., ‚ÄúCity eq 'London'‚Äù) in the ‚ÄòFilter‚Äô field to restrict the returned records to those that satisfy the condition.  See https://docs.oasis-open.org/odata/odata/v4.0/errata02/os/complete/part2-url-conventions/odata-v4.0-errata02-os-part2-url-conventions-complete.html#_Toc406398094[System Query Option $filter] in the OData documentation for more information about the `$filter` operator.
+
image:dataSourceOData7.png[alt="Filter field with filtering condition entered to restrict returned records"]

. Enter an expression in the ‚ÄòExpand‚Äô field to expand a *navigation property*.   See https://docs.oasis-open.org/odata/odata/v4.0/errata02/os/complete/part2-url-conventions/odata-v4.0-errata02-os-part2-url-conventions-complete.html#_Toc406398162[System Query Option $expand] in the OData documentation  for more information about the `$expand` operator.
+
TIP: A *navigation property* provides a link between records in different entities.   For example, if ‚ÄòOrders‚Äô is specified as the ‚ÄòExpand‚Äô field when ‚ÄòFilter‚Äô is set to ‚ÄúCustomerID eq 'ALFKI'‚Äù, this returns all the records in the ‚ÄòOrders‚Äô entity for customer ID ‚ÄúALFKI‚Äù.
+
image:dataSourceOData8.png[alt="navigation property set in Expand field to link entities"]

. Enter a comma-separated list in the ‚ÄòOrder By‚Äô field (e.g., ‚ÄúContactTitle,ContactName‚Äù) to order the records in the result set based on the specified columns.  See https://docs.oasis-open.org/odata/odata/v4.0/errata02/os/complete/part2-url-conventions/odata-v4.0-errata02-os-part2-url-conventions-complete.html#_Toc406398164[System Query Option $orderby] in the OData documentation  for information about the `$orderby` operator.
+
image:dataSourceOData9.png[alt="Order By field with comma-separated column list for sorting"]

. Enter an integer value in the ‚ÄòTop‚Äô field (e.g., ‚Äú5‚Äù, or ‚Äú0‚Äù to suppress filtering) to include only the specified number of initial records in the result set and omit the remaining records from the end of the result set.  See https://docs.oasis-open.org/odata/odata/v4.0/errata02/os/complete/part2-url-conventions/odata-v4.0-errata02-os-part2-url-conventions-complete.html#_Toc406398164[System Query Options $top and $skip]  in the OData documentation for information about the `$top` operator.


. Enter an integer value in the ‚ÄòSkip‚Äô field (e.g., ‚Äú5‚Äù, or ‚Äú0‚Äù to suppress filtering) to omit the specified number of initial records from the result set, and return the remaining records from the end of the result set. See https://docs.oasis-open.org/odata/odata/v4.0/errata02/os/complete/part2-url-conventions/odata-v4.0-errata02-os-part2-url-conventions-complete.html#_Toc406398164[System Query Options $top and $skip]  in the OData documentation for information about the `$skip` operator.
