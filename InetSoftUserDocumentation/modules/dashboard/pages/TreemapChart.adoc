Summary: Reference for creating a Treemap chart in Visual Composer, which displays a measure using proportionally sized rectangles arranged in a hierarchical layout. Place hierarchical dimensions in the T region and the measure in the Size region. TreemapChart, SunburstChart, IcicleChart, and CirclePackingChart all bind identically (T/Size/Color) and encode the same hierarchical part-to-whole data — they differ only in layout: Treemap uses nested rectangles, Sunburst uses radial wedges, Icicle uses layered rectangular bands (the Cartesian equivalent of Sunburst), and Circle Packing uses nested circles. MarimekkoChart shows a related but structurally different 2-level mosaic (X/Y/Color/Break By binding, not T/Size).
endif::ai[]

A treemap displays a *measure* by using the sizes of rectangles in a hierarchical display.

include::ROOT:partial$measureAndDimensionTip.adoc[]

image::commonscript:TreemapConfig1.png[alt="treemap chart example"]

To create a treemap chart, follow the basic steps below:

include::partial$chartNewbieTip.adoc[]

. Press the {buttonSelectChartStyle}. Choose the ‘Treemap’ style. Press the {buttonApply}.
+
image::commonscript:TreemapConfig2.png[alt="press Select Chart Style, choose Treemap"]

. From the ‘Dimensions’ folder of the *Data Source* panel, drag the desired hierarchical *dimension* fields to the ‘T’ region. Add dimensions from the outermost to the innermost level of the hierarchy.
+
include::ROOT:partial$dimensionTip.adoc[]
+
image::commonscript:TreemapConfigT.png[alt="drag hierarchical dimensions to T region"]

. From the ‘Measures’ folder of the *Data Source* panel, drag a desired *measure* to the ‘Size’ region.
+
include::ROOT:partial$measureTip.adoc[]
+
image::commonscript:TreemapConfigSize.png[alt="drag measure to Size region"]

. *_Optional:_* Drag a dimension from the *Data Source* panel to the ‘Color’ region to distinguish dimension values by color.
+
image::commonscript:TreemapConfig3.png[alt="drag dimension to Color region"]

. *_Optional:_* Press the ‘Edit Color’ button next to the dimension, and choose the desired colors to represent the dimension values. Press the {buttonApply}.
+
image::commonscript:EditColorButton.png[alt="press Edit Color button"]

. *_Optional:_* {multipleDimensionNoColor}

. {finishEditing}
+
image::commonscript:TreemapConfig1.png[alt="completed treemap chart"]

NOTE: To create a grid of charts, place dimensions into the ‘X’ and/or ‘Y’ regions as well. See xref:TrellisChartGrid.adoc[] for more information.
