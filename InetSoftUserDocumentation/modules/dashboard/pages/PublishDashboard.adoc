*Publishing a Dashboard* means making the Dashboard accessible to a wider audience, either inside or outside your organization.  There are several ways to do this, described below.

Make a Dashboard Accessible to Other Users::
If you are creating or editing a Dashboard in Visual Composer, and want to make the Dashboard visible to other users in the User Portal, do the following:

[square]
* Save it to a *public* folder in the Repository (i.e, not '`Private Dashboards`'). See xref:SavePublicDashboard.adoc[]. This will make the Dashboard available to other users who have READ permission for that folder.
* Ensure the *List on Portal Tree* option for the Dashboard is enabled. See xref:DashboardOptions.adoc[].

Control Who Can See or Edit a Dashboard::
To restrict which users, groups, or roles can view or modify a Dashboard, set repository permissions in Enterprise Manager. See xref:administration:SetRepositoryPermissions.adoc[].


Share a Dashboard via Email or Social Network::
To share a Dashboard with others via email, a link, or social networks, see xref:ShareDashboard.adoc[].  This is the best way to share a Dashboard with individuals (for example, outside the organization) who do not have access to the User Portal.

Prepare a Dashboard for Different Devices or Languages::
To configure device-specific layouts or multi-language localization, see xref:PrepareForPublication.adoc[].
