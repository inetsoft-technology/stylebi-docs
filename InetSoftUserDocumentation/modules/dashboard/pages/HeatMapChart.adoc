= Heat Map Chart

// metadata for AI
ifdef::ai[]
Reference for creating a Heat Map chart in Visual Composer, which displays a measure using a colored grid where cell color intensity represents the aggregated value at each dimension intersection. Place dimensions on X and Y axes, and the measure in the Color region. Do not place measures on X/Y axes and do not place any fields in the Size or Shape regions. A measure can optionally be placed in the Text region, but placing a dimension in the Text region will create a Wordcloud chart instead.
endif::ai[]

A heat map displays a *measure* using a colored grid, where the value of the aggregated measure for a given combination of *dimension* values is used to determine the color of the grid at that location.

include::ROOT:partial$measureAndDimensionTip.adoc[]

IMPORTANT: Do not place any measures on the ‘X’ and ‘Y’ axes, and do not place any fields in the ‘Size’ and ‘Shape’ regions of the Chart Editor. You can place a *measure* in the '`Text`' region  if desired. (Placing a *dimension* in the '`Text`' region will create a xref:WordcloudChart.adoc[] instead.)  {convertToMeasure}

image::commonscript:HeatMapConfig5.png[alt="heat map chart editor configuration"]

To create a heat map, follow the basic steps below:

include::partial$chartNewbieTip.adoc[]

. From the ‘Measures’ folder of the *Data Source* panel, drag a desired *measure* to the ‘Color’ region.
+
include::ROOT:partial$measureTip.adoc[]
+
image::commonscript:HeatMapConfigColor.png[alt="drag measure to Color region"]

. From the ‘Dimensions’ folder of the *Data Source* panel, drag a desired *dimension* to the ‘Y’ region.
+
include::ROOT:partial$dimensionTip.adoc[]
+
image::commonscript:HeatMapConfigY.png[alt="drag dimension to Y region"]

. From the ‘Dimensions’ folder of the *Data Source* panel, drag a desired *dimension* to the ‘X’ region.
+
image::commonscript:HeatMapConfig.png[alt="drag dimensions and measure to chart regions"]

. *_Optional:_* For a date dimension, press the {buttonEditDimension} next to the field name and set the ‘Level’ property to the desired date grouping. Press the {buttonApply}.
+
image::commonscript:HeatMapConfig2.png[alt="set Date dimension level to Month of Year"]

. *_Optional:_* {multipleDimensionNoColor}

. *_Optional:_* Press the ‘Edit Color’ button next to the measure, and choose the desired color spectrum to represent the aggregated values. (The built-in ‘Heat’ spectrum uses traditional shades of orange and brown, but you can use any other spectrum as well.) Press the {buttonClose}.
+
image::commonscript:HeatMapConfig3.png[alt="choose color spectrum for heat map"]

. {finishEditing}
+
image::commonscript:HeatMapConfig4.png[alt="completed heat map"]
+
TIP: Resize the chart to show the desired number of dimension values. To increase the density of the chart by using a finer grid, see xref:BasicChartingSteps.adoc#AdjustChartSpacing[Adjust Chart Spacing] in xref:BasicChartingSteps.adoc[].
