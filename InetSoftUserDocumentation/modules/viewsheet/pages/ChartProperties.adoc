To open the  ‚ÄòChart Properties‚Äô panel, right-click body of the chart, and select ‚ÄòProperties‚Äô from the context menu. {rightClickOptions}  The ‚ÄòChart Properties‚Äô panel provides the following tabs: *General*, *Advanced*, *Line*, *Hierarchy*, *Script*.

[#GeneralTab]
== General Tab

The *General* tab in the ‚ÄòChart Properties‚Äô panel provides the following Chart-specific properties.

NOTE: {dynamicProperties}



Tip::
* *Tooltip*: Value is displayed as a tooltip when the mouse hovers over the corresponding chart area. To display custom HTML in the tooltip, see xref:AddTipsToChart.adoc[].
* *Data Tip View*: A Dashboard component is displayed when the mouse hovers over the chart area. The component is filtered based on the hover region. Select ‚ÄòOn Click Only‚Äô if you want the data tip to appear only when the user clicks (rather than hovers). If the clicked element has a hyperlink, you must {buttonCtrl}-click to show the data tip. See xref:AddTipsToChart.adoc[] for more details.
* *Flyover Views*: Allows you to select a set of Dashboard components which will be adaptively filtered as the user hovers the mouse or clicks on the chart area. (Select ‚ÄòOn Click Only‚Äô if you want the filtered components to update when the user clicks.)  See  xref:DrillDownIntoData.adoc[] for more details.

== Advanced Tab
The *Advanced* tab in the ‚ÄòChart Properties‚Äô panel provides the following properties. The available properties depend on the Chart type; some properties are not available for certain chart types.  (See xref:BasicChartingSteps.adoc[] for information on how to select the Chart type.)

=== Chart Options
Glossy Effect:: Applies a 3D effect to the Chart as a whole. (Not available for all Chart styles.)
Sparkline:: Hides all axes and legends. For a line chart, the ‚ÄòSparkline‚Äô also adds a point to the end of the line, and for a bar chart, displays negative values in red.
Enable Ad Hoc Editing:: Allow end users to access the Chart Editor and edit the chart.
Enable Date Comparison:: Enables the Date Comparison feature for the Chart. See xref:CompareDataByDate.adoc[] for more details.
Enable Drilling:: Allows the user to drill up and down in date or dimension fields. See xref:DrillDownIntoData.adoc[] for more information.
Sort Others Last:: Available when ‚ÄòGroup all others together‚Äô is enabled for Top/Bottom ranking in the ‚ÄòEdit Dimension‚Äô panel of the Chart Editor. (See xref:FilterTopBottomGroups.adoc[]  for information about dimension ranking). When enabled, this property places the ‚ÄòOthers‚Äô group _after_ all of the ranked groups, regardless of the specified dimension sorting.
Top N For Each {...}:: When enabled, specifies that the top-N filter for a dimension (see xref:FilterTopBottomGroups.adoc[]) should be relative to the parent dimension in a dimension hierarchy. (See xref:#HierarchyTab[] for explicit hierarchy.)
+
****
*Example*: If the chart ‚ÄòX‚Äô dimension is ‚ÄòProduct Category‚Äô and the ‚ÄòColor‚Äô binding is ‚ÄòProduct Name‚Äô, and you request ‚Äútop 3‚Äù of ‚ÄòProductName‚Äô, enable ‚ÄòTop N For Each Product Category‚Äô to make the filter relative to the ‚ÄòProduct Category‚Äô grouping (yielding the top 3 products for each category). Leave the option disabled to filter ‚ÄòProduct Name‚Äô without respect to the ‚ÄòProduct Category‚Äô group.
****


=== Plot
Background:: The fill color of the plot area on the Chart.
+
TIP: To make the Chart background fully *transparent* (so that the Dashboard background is visible through the Chart), you should make the plot  '`Background`'  transparent here, and also make the component background transparent.  See xref:VisualFormat.adoc[] for more information.

Border Color:: For xref:CirclePackingChart.adoc[], sets the border color for the circles.

Pareto Line Color:: The color of the cumulative total line when a field is bound in the ‚ÄòColor‚Äô region. If there is no ‚ÄòColor‚Äô binding, the Pareto line is the same color as the bars.
Alpha:: Sets transparency for the Chart elements. If disabled, the default transparency for the particular chart type is used.

=== Banding
Banding:: Adds alternating colored bands to a specified categorical or timescale axis. Select the desired band color from the color picker. The band ‚ÄòSize‚Äô setting is an integer indicating the number of category groups (for a categorical axis) or the number of date groups (for a timescale axis) to be spanned by each band.

=== Plot Options (General)
The following plot options are typically available for all Chart types.

Show Values:: Displays the aggregate measure values on the chart. If the chart also possesses a ‚ÄòText‚Äô binding, then the values displayed by ‚ÄòShow Values‚Äô and those displayed by the ‚ÄòText‚Äô binding appear on different lines. Use a ‚ÄòText‚Äô format to individually format the values. For example, if the top value represents a date and the bottom value represents a number, you can apply the following text format, using `\n` to separate lines:
+
 Date: {0,date,yyyy}\nAmount: {1,number,$#}
+
For a *Map-type* chart, ‚ÄòShow Values‚Äô displays the geographic labels for all regions if there is no ‚ÄòText‚Äô binding.
+
NOTE: By default, measure values are also displayed as a *tooltip* when the user *hovers* the mouse over the chart. Tooltips are visible when the Dashboard is previewed or viewed in the Portal. See xref:#GeneralTab[] to customize the tooltip.

Show Reference Line:: Displays horizontal and vertical reference lines when the user moves the mouse over a data point.

Stack Value:: Displays a single aggregate value for each primary (X-axis) grouping, rather than for each individual subseries grouping.

Keep Element in Plot:: Automatically extend the chart boundaries to display chart elements (including target lines) that exceed the specified axis maximum.

Fill Time-Series Gaps:: When the ‚ÄòAs time series‚Äô option is set for a dimension, the ‚ÄòFill Time-Series Gaps‚Äô option specifies how missing data should be represented on the chart. (See xref:BasicChartingSteps.adoc[] for information on how to edit a dimension.) When the option is disabled, plotted lines simply ignore the missing data and connect adjacent points. When the option is enabled, you can choose to represent gaps with either ‚ÄòNull‚Äô, which leaves an empty space at the location, or with ‚ÄòZero‚Äô, which inserts a numerical value of 0 at the location.

Stack Measures:: Stacks multiple chart measures together. This may be desirable when the individual measures are fractions of some total. (Example: Component fees on a shipment, such as documentation, packaging, handling, etc.) This option is available only when there are multiple measures on the chart, the ‚ÄòStack‚Äô option is enabled from the {buttonSelectChartStyle}, and single-graph mode has been activated from the {buttonSwitchSingleGraph}.

=== Plot Options (Pie Chart)
The following plot options are  available for xref:viewsheet:PieChart.adoc[] types.

Explode Pie:: Displays a pie chart with small gaps between the slices.

Pie Ratio:: A value in the range [0.1,1] that determines the proportion of the plot area occupied by the pie. Similarly-sized Pie charts with the same ‚ÄòPie Ratio‚Äô setting will display pies having the same radius, which allows you to enforce consistency across multiple charts.
+
CAUTION: Large  ‚ÄòPie Ratio‚Äô settings may leave less space for the display of values on the Chart.

=== Plot Options (Bar Chart)
The following plot options are available for xref:viewsheet:BarChart.adoc[] types (non-stacked, non-3D only).

Corner Radius:: A value between 0 and 0.5 that controls the rounding of bar corners. A value of `0` produces square corners; `0.5` produces the maximum rounding. Only the open (value) end of each bar is rounded unless 'Round All Corners' is also enabled. New charts default to a radius of 0.3.

Round All Corners:: When enabled, rounds all four corners of each bar equally using the 'Corner Radius' value. When disabled (default), only the open (value) end of each bar is rounded.

=== Plot Options (Network/Tree Chart)
The following plot option is  available for xref:viewsheet:NetworkChart.adoc[], xref:CircularNetworkChart.adoc[] and xref:TreeChart.adoc[] types.

Apply Aesthetics to Source Nodes:: Specifies that node ‚ÄòColor‚Äô and ‚ÄòSize‚Äô determined by ‚ÄòTarget‚Äô value should also be applied to source nodes that are not also target nodes.

Smooth Lines:: For xref:CircularNetworkChart.adoc[] only, when enabled (default), draws chart edges as smooth B√©zier curves instead of straight lines.


=== Plot Options (Circle Packing/ Treemap)
The following plot option is  available for xref:viewsheet:CirclePackingChart.adoc[] and xref:TreeChart.adoc[] types.


Include Parent Labels:: This includes the parent category names in the circle labels.

=== Plot Options (Word Cloud Chart)
The following plot option is available for xref:viewsheet:WordcloudChart.adoc[] types.


Font Scale:: Multiplies the specified ‚ÄòSize‚Äô setting to increase the maximum font size of words in the cloud.

=== Plot Options (Line Chart)
The following plot options are  available for xref:viewsheet:LineChart.adoc[], xref:viewsheet:AreaChart.adoc[], and xref:RadarChart.adoc[] types.

Show Points:: Adds markers at the locations of the individual data points.

Smooth Lines:: When enabled, renders chart lines as smooth curves instead of straight segments. Enabled by default for Area charts; disabled by default for Line charts.  Not available for Radar Charts.

Fill Missing Data with Dashed Line:: Linearly interpolates missing data using a dashed line.



=== Plot Options (Point Chart)
The following plot options are  available for xref:viewsheet:PointChart.adoc[] types.


Show Lines:: Displays line segments connecting the points on the chart. To display a fitted line, use the ‚ÄòTrend Line‚Äô option instead. (See xref:#LineTab[].)

As One Line:: When a dimension is set in the *Color*, *Shape*, *Size*, or *Text* regions, connects the points with a *single line* rather than using separate lines for the different dimension values. Requires ‚ÄòShow Lines‚Äô to be enabled.

Fill Missing Data with Dashed Line:: When ‚ÄòShow Lines‚Äô is enabled, linearly interpolates missing data using a dashed line.


=== Plot Options (Map Chart)
The following plot options are  available for xref:viewsheet:MapChart.adoc[]  types.


Map Default Color:: Sets the color for map regions that are not assigned a color based on data.

Always Show Color in Map:: When a column is bound to the ‚ÄòColor‚Äô field of the *Visual* panel in the Chart Editor, this setting ensures that the color is always used to fill the map regions. Otherwise, if another column is simultaneously bound to the ‚ÄòShape‚Äô, ‚ÄòSize‚Äô, or ‚ÄòText‚Äô fields of the *Visual* panel, the color generated for the ‚ÄòColor‚Äô binding will be applied to the plotted points rather than to the map regions.

=== Plot Options (Contour Map/Scatter Contour)
The following plot options are  available for xref:viewsheet:ContourMapChart.adoc[] and xref:ScatterContourChart.adoc[] types.

Levels:: Specifies the number of contours used to represent each region. A larger number produces a smoother transition of colors.

Bandwidth:: Specifies the area over which to calculate density. A larger number produces contours that span larger areas.

Edge Alpha:: Specifies the transparency of the contour edge. The transparency of the contour center is determined by the  Chart ‚ÄòAlpha‚Äô property. A higher value makes the edges more visible.

Cell Size:: Specifies the size of the cells that make up the grid on which contour weights are computed. Higher values specify larger cells, and hence broader clusters. Lower values specify smaller cells, and hence smaller clusters.

=== Plot Options (Bar Chart/Interval Chart)
The following plot options are  available for xref:viewsheet:BarChart.adoc[] and xref:IntervalChart.adoc[] types.

Bar Corner Radius:: Sets the radius for rounding the bar corners. The range is 0.0 (no rounding) to 0.5 (maximum rounding). The default is 0.3.
Round All Corners:: For xref:BarChart.adoc[], specifies (when enabled) that the '`Bar Corner Radius`' should be applied to all four corners of the bar.  When disabled (default), only the corners furthest from the axis are rounded.  This option is always *enabled* for xref:IntervalChart.adoc[].

=== Target Lines
Specify marker lines, bands, or statistical regions to be placed in the plot area. See xref:TrendCompareData.adoc[] for more information.

[#LineTab]
== Line Tab
The *Line* tab in the ‚ÄòChart Properties‚Äô panel provides the following chart-specific properties. The properties shown depend on the chart style.  See xref:BasicChartingSteps.adoc[] for information on how to set the Chart style.

image::Line Chart Tab.png[alt="chart properties line tab"]

X Grid:: The line style and color of the X-axis grid.
Y Grid:: The line style and color of the Y-axis grid.
Quadrant Grid:: The style and color for a set of perpendicular line segments which divide the plot area into quadrants.
Diagonal Line:: The style and color for a line segment which originates in the lower-left corner of the plot area and terminates in the upper-right corner of the plot area.
Show Table Grid:: For a facet-type chart (i.e., a chart with both a *measure* and *dimension* on the same axis or with multiple dimensions on the same axis), displays a solid line around the entire plot area and displays a solid axis line to separate the dimension and measure axis labels.  See xref:HybridTableChart.adoc[] for an example of this kind of Chart.
+
include::ROOT:partial$measureAndDimensionTip.adoc[]
+
Use the adjacent color chip to select the color for the line surrounding the plot area. To set the color of the axis separator line, right-click on the line and select ‚ÄòAxis Properties‚Äô from the context menu. See  xref:AxisProperties.adoc[] for details about setting the axis color.
Trend Line:: Specifies whether an interpolated trend line is displayed, and the method by which the trend line should be fit to the data (linear, quadratic, etc.). The ‚ÄòTrend Line Style‚Äô menu and color chip specify the trend line style and color. The ‚ÄòDefault‚Äô option indicates that the style should match the corresponding measure line. If there are multiple *measures* or if there is a *dimension* associated with the chart‚Äôs ‚ÄòColor‚Äô binding, select the ‚ÄòOne Per Color‚Äô option to create an independent trend line for each color group. (The trend line colors are matched to the corresponding data colors unless you explicitly specify a trend line color.)
+
include::ROOT:partial$measureAndDimensionTip.adoc[]
+
Use the ‚ÄòFor Measures‚Äô menu to select a subset of measures for which to display trend lines.

Project Forward:: Specifies the number of axis units that the trend line should extend beyond the final data point.  Projection requires that axis values be a linear and complete series of either numbers or dates.
[square]
* *Not available* for the following chart types: Waterfall, Pie, Radar, Filled Radar, Map.
* *Not available* for the following data types: String, Boolean.

The *Hierarchy* tab in the ‚ÄòChart Properties‚Äô dialog box allows you to specify the hierarchy for a custom drill-down. To create a custom drill-down hierarchy, follow the steps below:




. Right-click the Chart, and select ‚ÄòProperties‚Äô from the context menu. {rightClickOptions} This opens the ‚ÄòChart Properties‚Äô panel.

. Select the *Hierarchy* tab. The ‚ÄòColumns‚Äô panel displays the fields available in the Chart‚Äôs binding.
+
image::Hierarchy Chart Tab.png[alt="select hierarchy tab"]

. Click on a field that you want to include in the hierarchy, and drag the field to the ‚ÄòHierarchy‚Äô panel.


. *_Optional:_*  If the added field is a date field, select the desired date representation from options below the ‚ÄòHierarchy‚Äô panel.
+
TIP: Drilling on date fields is enabled by default. Use the *Hierarchy* tab to _customize_ the drill sequence, as shown below.

+
image::Chart Properties Add Hierarchy3.png[alt="add date field to hierarchy"]
+
After you add a date field to the ‚ÄòHierarchy‚Äô panel, the date field still remains available in the ‚ÄòColumns‚Äô panel. This allows you to add the same date field to the hierarchy at multiple levels, giving each level a different date representation. In this way, you can control the date-drilling sequence. For example, the following hierarchy produces the drill-down sequence menu:Year[Month > Day].
+
image::Chart Properties Add Hierarchy4.png[alt="date field hierarchy drill sequence"]

. Click on another field that you want to include in the hierarchy, and drag it over the existing fields in the ‚ÄòHierarchy‚Äô panel. Drop the field into the desired position in the hierarchy.
+
image::Chart Properties Add Hierarchy.png[alt="drag field into hierarchy panel"]

. Repeat the previous step to add additional fields to the hierarchy, as desired.

. To remove a specific field from the hierarchy, simply drag the field from the ‚ÄòHierarchy‚Äô panel back to the ‚ÄòColumns‚Äô panel. To remove _all_ fields from the hierarchy, press the btn:[Clear] button.

. *_Optional:_*  To create an additional independent hierarchy, drag a field from the ‚ÄòColumns‚Äô panel to the region labeled ‚ÄòDrag here to add a hierarchy‚Äô. Then repeat the previous steps to add additional fields to this independent hierarchy.
+
image::Chart Properties Add Hierarchy2.png[alt="add independent hierarchy"]

. Press btn:[OK] to close the panel.

Note that there may not be any immediate change to the Chart. However, by configuring a hierarchy as described above,  when you (or the end user) adds a field listed on the hierarchy to the ‚ÄòX‚Äô or ‚ÄòY‚Äô region of the Chart Editor, the Chart will automatically implement an appropriate drill-down feature based on the specified hierarchy.

.Example: Dimension  Hierarchy
****
Consider the following hierarchy: menu:Category[Name>Company]. After you have specified this hierarchy in the ‚ÄòHierarchy‚Äô panel, if you then bind the ‚ÄòCategory‚Äô field to the ‚ÄòX‚Äô or ‚ÄòY‚Äô region in the Chart Editor, the Chart automatically makes a full drill-down hierarchy (menu:Category[Name>Company]) available to users. Likewise, if you bind the ‚ÄòName‚Äô field to the ‚ÄòX‚Äô or ‚ÄòY‚Äô region in the Chart Editor, the Chart automatically makes the appropriate partial drill-down hierarchy (menu:Name[Company]) available to users.
****
