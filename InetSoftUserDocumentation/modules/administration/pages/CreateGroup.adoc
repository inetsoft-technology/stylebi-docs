A group is a collection of users, such as a project team, a department, or a geographical location. (You can create and edit groups in Enterprise Manager only when using the ‚ÄòDefault‚Äô authentication provider.) Groups are generally organizational in nature, while roles are functional in nature.

== Create a New Group
To create a new group, follow the steps below:


. Press the {buttonSettings} at the top of Enterprise Manager.
. Select the *Security* page in the left panel.
+
image::Figure 23_1.png[alt="select Security page in left panel"]
. Select the *Users* tab.
+
image::securityDefineUsers.png[alt="select Users tab in Security page"]

. From the ‚ÄòProvider Name‚Äô menu, select the security provider under which to define groups.

. Press the ‚ÄòNew Group‚Äô link below the tree. This creates a group with the default name ‚ÄògroupN‚Äô (group0, group1, etc.).
+
image::server3.png[alt="new group created in security tree"]

. Type in the ‚ÄòName‚Äô field to change the name of the group.

. To select a Portal theme for the group, choose the desired theme from the ‚ÄòTheme‚Äô menu. (See xref:Presentation.adoc[] for information about how to create a theme.)

+
IMPORTANT: The multi-tenant feature in Enterprise Edition must be *disabled*.  See xref:EnableMultiTenant.adoc[] for more information.


. *_Optional:_* To add the users or groups to a group, press the btn:[Add] button in the ‚ÄòMembers‚Äô area. Choose the desired group(s) in the ‚ÄòAdd Group/User‚Äô dialog box (kbd:[Ctrl]-click to select multiple groups or users), and press btn:[Add].
+
image::securityDefineGroup.png[alt="add members and roles to group dialog"]

. *_Optional:_* To add a role for users in the group, press the btn:[Add] button in the ‚ÄòRoles‚Äô area. Choose the desired role(s) in the ‚ÄòAdd Role‚Äô dialog box (kbd:[Ctrl]-click to select multiple roles), and press btn:[Add].

. *_Optional:_* To add administrative permission for the group (ability for other users to administer this group), press the btn:[Add] button in the ‚ÄòAdministrator Permissions‚Äô area. Choose the desired user(s), groups(s), or role(s) in the ‚ÄòAdd Permission‚Äô dialog box (kbd:[Ctrl]-click to select multiple items), and press btn:[Add].
Users, groups, or roles granted such administrative privileges will be able to use Enterprise Manager to remove the group or reassign users and roles to the group.
. Press {buttonEMApply} to save your changes.
. Repeat the above steps to add additional groups.

== Copy and Paste Identities

IMPORTANT: You cannot copy and paste identities across different organizations or different security providers.

To copy the '`Members`' or '`Roles`' identities from one group to another, follow the steps below:

. Press the {buttonSettings} at the top of Enterprise Manager.
. Select the *Security* page in the left panel.
+
image::Figure 23_1.png[alt="select Security page in left panel"]
. Select the *Users* tab.
+
image::securityDefineUsers.png[alt="select Users tab in Security page"]

. From the ‚ÄòProvider Name‚Äô menu, select the security provider under which to define groups.

include::partial$selectOrganization.adoc[]


. Expand the ‚ÄòGroups‚Äô node on the tree, and select the desired group.

. Press the btn:[Copy Identities] button below the '`Members`',  '`Roles`', or '`Administrator Permissions`' tables to copy the current identities. You can then navigate to another group and paste those identities.

. Press the btn:[Paste Identities] button below the corresponding table to paste the identities you have copied.
+
TIP: The btn:[Paste Identities] button will display numbers in parentheses, such as btn:[Paste Identities (x of n)]. The *second value* (*n*) indicates the number of identities that were copied to the clipboard. The *first value* (*x*) indicates the number of identities that are compatible with the current group, and can actually be pasted.

. Press {buttonEMApply} to save your changes.
